32//! ## Motivation
33//!
34//! I have a few Rust projects that are compiled for Linux and Windows. I need to save some paths inside a config file. Windows have a strange way to work with file/folder paths. I need a library to work in a neutral crossplatform way. Only at the last line of code I transform the neutral path into something that the current OS recognizes.
35//!
36//! There exist already some libraries for that: relative-path, typed-path, x-path, camino,...
37//!
38//! But it is fun to make something new and simple and very very opinionated.
39//!
40//! ## Opinionated to the max
41//!
42//! My opinions are probably not useful for all developers, but they work for me and probably for most.
43//!
44//! 1. The path will be strictly utf8. I know that there exists a rare possibility of the path being in some other strange format, but I will never encounter that with this library. Or at least, I will always avoid that.
45//! 2. Unix and Linux have paths that look nice. Windows is the problem here. The crossplatform format will be very much like the Linux paths.
46//! 3. The only path separator will be the universal '/'. If some '\\' exists, it will be replaced by '/'. Linux allows the use of '\\' inside a filename, but my opinion is that this is bad and should be avoided.
47//! 4. Linux is very permissive. Only NULL and '/' are forbidden in path components. But it is a good idea to not allow special characters forbidden on Windows:  
48//! < > : " / \\ | ? *  
49//! 0 (NULL byte)  
50//! 0-31 (ASCII control characters)  
51//! 5. Filenames cannot end in a space or dot.
52//! 6. Not allow reserved filenames even with extensions:
53//! CON, PRN, AUX, NUL  
54//! COM1, COM2, COM3, COM4, COM5, COM6, COM7, COM8, COM9  
55//! LPT1, LPT2, LPT3, LPT4, LPT5, LPT6, LPT7, LPT8, LPT9  
56//! .  (special name referring to current directory)  
57//! .. (special name referring to parent directory)  
58//!
59//! 7. Instead of the problematic Windows 'c:' or 'd:' drives, the neutral crossplatform format will be '/mnt/c' or '/mnt/d'  
60//! 8. This special symbols are allowed and will be transformed for Windows:  
61//! '~'    will be transformed into %UserProfile%
62//! /tmp   will be transformed into %TEMP%
63//! /mnt/c/ will be transformed into c:\\
64//! /mnt/d/ will be transformed into d:\\
65//!
